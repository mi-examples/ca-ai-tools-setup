# Cursor — agent skills

Skills under **`.cursor/skills/`** guide **Cursor Agent** for Metric Insights **Portal Page / Custom App** work. **Claude Code** uses mirrored copies under **`.claude/skills/`** for core dev/test skills (see **`README.md`** there) and **`.claude/workflows/`** for full QA orchestration.

## Skills

- **`ai-development`** — Linear CLI + dev gate, greenfield pp-dev, Figma MCP, PR review, DoD + **`DOD-FULL.md`**.
- **`ai-testing`** — Linear CLI + test gate, UI check (screenshots in comments), manual test-documentation format.
- **`figma-code-connect`** — Figma **Code Connect** template authoring (`.figma.ts`) via Figma MCP, plus **`references/`**. _(Present only when this repo was bootstrapped with Figma MCP.)_
- **`testing-flow`** — full QA flow (Linear optional).
- **`testing-with-linear`** — Linear-driven QA orchestration.
- **`ui-check`** — focused UI verification against a Linear issue.
- **`ui-check-simple`** — quick UI check without Linear.
- **`linear-report`** — publish QA results to Linear.
- **`playwright-mcp`** — Playwright MCP tools reference.
- **`linear-workflow`** — branch QA/Linear workflow helper for legacy flows.
- **`test-documentation`** — branch manual TC/bug documentation helper.
- **`figma-implementation`** — Figma MCP design reference for visual QA (optional in testing flows).
- **`form-builder`** — Form Builder / schema guidance for MI backend forms.

## Rules

Structured Cursor rules live under **`.cursor/rules/*.mdc`** (Linear gates, CLI, env, test docs, Figma implementation). Read **`README.md`** in that directory.

## Notes

- Generated by **`@metricinsights/ca-ai-tools-setup`**. Re-run with **`--force`** to refresh after upgrading the bootstrap package.
- When you change a skill, update **both** `.cursor/skills/...` and `.claude/skills/...` (or workflows), or re-run the installer.
